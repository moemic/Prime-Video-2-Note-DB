---
description: タスク完了時に自動的にGitでバックアップ（Push）を行うワークフロー
---

機能の追加や修正が完了し、`walkthrough.md` を作成または更新した直後に、以下の手順を自動的に実行します。

1. **CHANGELOG.md の更新**
   今回行った変更内容を `CHANGELOG.md` の冒頭（最新セクション）に追記する。

2. **変更の確認**
   `git status` を実行し、未コミットの変更があるか確認する。

2. **ステージング**
   `git add .` を実行する。

3. **コミット**
   タスクの内容（Task NameやSummary）に基づいた具体的なコミットメッセージを作成してコミットする。
   メッセージ例: `fix: タイトル合成機能の削除とコードのクリーンアップ`

4. **プッシュ**
   // turbo
   `git push origin main`（または現在のブランチ）を実行する。

// turbo-all
