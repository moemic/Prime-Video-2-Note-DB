# Notion画像完全入れ替え計画

## 目的

既存ページで「カバー画像を上書きする」を選んだ場合、古い画像を残さず、今回選択した画像を先頭にした画像一式へ完全に入れ替える。

## 実装内容

1. 既存ページかつ上書きチェックありの場合を「画像完全入れ替え」として送信する。
2. 完全入れ替え時は既存の「カバー画像」Filesプロパティを引き継がない。
3. クリック選択画像をFiles配列の先頭にし、残りの取得画像を後ろへ並べる。
4. 右クリック選択画像はNotionページ上部のカバーとして設定する。
5. 新規画像が0枚でも、完全入れ替え時はFilesプロパティを空配列にして古い画像を削除する。
6. 通常更新では従来どおり既存画像を維持する。
7. バージョンを `1.32.0` に更新し、変更履歴を記録する。

## 検証

- `npm run check:image-replacement`
- `node --check background.js`
- `node --check popup.js`
- `npm run check:version`
- `git diff --check`
