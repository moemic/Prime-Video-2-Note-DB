# 修正内容の確認 (Walkthrough) - ステータスUIの刷新

ステータス欄のUIを「タグ」欄と同じチップ形式に変更し、操作性を向上させました。

## 変更内容の概要

### 1. チップ形式のステータス選択
- ステータス候補がボタン（チップ）として一覧表示されるようになりました。
- クリックするだけでステータスを切り替えられます（選択中のステータスはハイライト表示されます）。
- 選択されたステータスは入力欄の中にチップとして表示され、`×` ボタンで解除可能です。

### 2. 単一選択の維持
- UIはタグと似ていますが、ステータスは常に「1つのみ」選択されるように制御されています。別の候補をクリックすると、以前の選択は自動的に置き換わります。

### 3. 引き続き自由入力も可能
- 候補にないステータスを設定したい場合は、入力欄にテキストを入力して `Enter` キーを押すことで設定可能です。

### 4. バグ修正 (v1.11.0)
- ステータス候補取得時に `options.forEach is not a function` となる不具合を修正しました。
- Notion側が「セレクト」型の場合に「ステータス」型として保存しようとしてエラーになる不具合を修正しました。
- 拡張機能更新直後に「抽出エラー」が発生した場合、ページのリロードを促す案内を表示するようにしました。

### 5. バージョン更新 (v1.11.0)
- 不具合修正を反映し、バージョンを **v1.11.0** に更新しました。
